How to Run Test Programs on Unix

Last Updated 09/26/03

Test programs are provided to show how to develop client applications and providers using the WBEM Services client and provider APIs. Test client programs can be found in the examples/client subdirectory where the WBEM Services executables are located. Test provider programs are found in subdirectory examples/provider. To run some of these example programs to illustrate how the client and provider APIs work, execute the run_tests.sh script.

  • First start the CIMOM by following the directions found here.
  • Go to the directory containing the WBEM Services executables.
    • If you built the WBEM Services executables in /files/open, go to directory /files/open/dist/wbemservices/examples.
    • If you downloaded or copied the WBEM Services executables into /wbemservices, go to directory /wbemservices/examples.
  • Run sh run_tests.sh. You will see the following: 
     wbem% cd /files/open/dist/wbemservices/examples
 wbem% sh run_tests.sh
USAGE: sh run_tests.sh user password protocol [host]
   where protocol is rmi (the default) or http

  • run_tests.sh runs test programs to illustrate how some client APIs interact with the CIMOM. It also runs programs to illustrate how client APIs interact with test providers. In these cases, test classes are added to the CIM Repository as needed. These classes are later removed before run_tests.sh terminates.

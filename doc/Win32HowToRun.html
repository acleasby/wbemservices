How to Install and Run WBEM Services on Win32

Last Update 09/26/03

To run the WBEM Services executables on Win32

  • Open a command window (DOS window).
  • Go to the directory containing the WBEM Services executables.
    • If you built the WBEM Services executables in c:\files\open, go to directory c:\files\open\dist\wbemservices\examples.
    • If you downloaded or copied the WBEM Services executables into c:\wbemservices, go to directory c:\wbemservices\examples
  • set JAVA_HOME=<path to JDK>. If not set, c:\jdk1.4.1_02 is used by default.
  • Run start_cimom.bat. The CIMOM will start but the batch file will not return to the command prompt.
  • Open a second command window (DOS window). Perform the following operations in the second window.
  • set JAVA_HOME=<path to JDK>. If not set, c:\jdk1.4.1_02 is used by default.
  • To verify that the CIMOM is operating properly, run verify_cimom.bat. This script requests information for CIM_Collection over HTTP and CIM_LogicalElement over RMI. To reference a different class, edit verify_cimom.bat.
  • To stop the CIMOM, type Control c in the first command window and answer "yes" if there is a Terminate batch job (Y/N)? prompt.

To install the WBEM Services executables on another system

  • Copy the wbemservices directory and sub-directories to the other system, or
  • Extract wbemservices.zip to the other system

To enable debug output of the server

Modify cimom/bin/WBEMServices_Unix.properties, and set DebugLevel=3.

To enable XML tracing of client applications

When invoking a client program, set system property wbem.debug.xml to 3 (e.g. -Dwbem.debug.xml=3)

The XML trace will be created/appended to file CIMClientXML_Trace.txt in the directory in which you run the client application. The XML trace will also be written to stdout.

To enable debug output of the server:

Modify cimom/bin/WBEMServices_Win32 or When invoking a client program, set system property wbem.debug.xml to 3 (e.g. -Dwbem.debug.xml=3)

To view WBEM Services documentation:

The following documentation is available to view using a browser:

  • WBEM Services API in JavaDoc form. In the top left frame, select the package you wish to view (e.g. javax.wbem.client). Then, in the lower left frame, select the class you wish to view (e.g. CIMClient). The right frame displays the following information about this class:
  1. Class description
  2. Summary tables for Fields, Constructors, and Methods (i.e. the APIs)
  3. Detailed information about each Constructor and Method (i.e. API)
  • WBEM Services MOF files in JavaDoc form. In the top left frame, you can select to view in the lower left frame, a list of all classes or only association classes. Then, in the lower left frame, select the MOF class you wish to view (e.g. CIM_Account). The right frame displays the following information:
  1. MOF Class description
  2. Superclass heirarchy
  3. List of known subclasses
  4. Table of class qualifiers
  5. Table of properties that are locally defined in the MOF class. Each entry shows the property name, its description, its data type, and any other property specific qualifier information provided in the MOF file.
  6. List of inherited properties. Each entry shows the property name, its data type, and its class origin (i.e. the class where the property was originally defined).
  • WBEM Services MOF classes in JavaDoc form. To view the MOF class heirarchy, select TREE in the right frame. To view as a full frame, select NOFRAMES

For further information, go to the WBEM Services website.

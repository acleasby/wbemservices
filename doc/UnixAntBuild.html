How to Build WBEM Services Executables using ANT on Unix

Last Updated 09/26/03

Introduction

ANT is a Java based build tool. This document gives instructions on how to build WBEM Services executables from the source code using ANT.

Requirements

To build executables from the WBEM Services source code using ANT, you first need to do the following:

  • Install Java2 SDK and JRE (Standard or Enterprise Edition) into /usr/java onto your system, version 1.4.1 (NOTE: Do not use 1.4.1_03 or higher or 1.4.2)
  • Install the ANT binaries onto your system, version 1.4; e.g /usr/jakarta-ant-1.4.
  • If needed, give execute permission to ANT files in the bin subdirectory; e.g. chmod +x /usr/jakarta-ant-1.4/bin/*
  • Checkout the WBEM Services source code from the CVS Repository

Instructions for Creating WBEM Executables using ANT on Unix

Assuming that the top directory of your WBEM Services source tree is /files/open, to create and package the WBEM Services executables, do the following:

  • If your JDK is NOT installed in /usr/java, then set the JAVA_HOME variable; e.g export JAVA_HOME=/files/j2sdk1.4.0
  • If ANT is NOT installed in /usr/jakarta-ant-1.4, then set the ANT_HOME variable; e.g export ANT_HOME=/files/jakarta-ant-1.4
  • cd /files/open
  • ./build.sh
  • The executables can be found in subdirectories under /files/open/dist/wbemservices

How to Install and Run WBEM Services on Unix

Last Update 09/26/03

To run the WBEM Services executables on Unix

  • Run ps -eaf | grep java to check whether a CIMOM is already running on your system. If so, as root user run kill -9 <cimom_pid> to terminate it.
  • Go to the directory containing the WBEM Services executables.
    • If you built the WBEM Services executables in /files/open, go to directory /files/open/dist/wbemservices/examples.
    • If you downloaded or copied the WBEM Services executables into /wbemservices, go to directory /wbemservices/examples.
  • As root user, run sh start_cimom.sh to start the CIMOM.
  • To verify that the CIMOM is operating properly, run sh verify_cimom.sh. This script requests information for CIM_Collection over HTTP and CIM_LogicalElement over RMI. To reference a different class, edit verify_cimom.sh.
  • To stop the CIMOM, run sh stop_cimom.sh and follow the instructions.

To install the WBEM Services executables on another system

  • Copy the wbemservices directory and sub-directories to the other system, or
  • Extract wbemservices.zip to the other system

To enable debug output of the server

Modify cimom/bin/WBEMServices_Unix.properties, and set DebugLevel=3.

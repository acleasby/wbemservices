How to Run CIM Workshop on Win32

Last Updated 09/26/03

CIM Workshop provides a graphical user interface that you can use to browse the CIM Schema and to perform operations on CIM Classes and Instances. To run this application:

  • First start the CIMOM by following the directions found here.
  • Open a command window (DOS window).
  • Go to the directory containing the WBEM Services executables.
    • If you built the WBEM Services executables in c:\files\open, go to directory c:\files\open\dist\wbemservices\examples.
    • If you downloaded or copied the WBEM Services executables into c:\wbemservices, go to directory c:\wbemservices\examples
  • To start CIM Workshop, run cimworkshop.bat. You will see the following: 

    c:>cd c:\files\open\dist\wbemservices\bin
c:\files\open\dist\wbemservices\bin>cimworkshop
*********************************************************
Starting up cimworkshop
*********************************************************

c:\jdk1.2.2\bin\java -classpath c:\files\open\dist\wbemservices\lib;c:\files\open\dist\wbemservices\bin\cimworkshop.jar;c:\files\open\dist\wbemservices\lib\wbem.jar -Dinstall.dir=c:\files\open\dist\wbemservices\bin org.wbemservices.wbem.apps.cimworkshop.CIMWorkshop
  • Next, a Login window will appear. By default, cimworkshop will communicate with the CIMOM over RMI. If you want to use HTTP, change the Transfer protocol from RMI to HTTP by clicking on the Down arrow and selecting HTTP.
  • Enter values for User Name and Password (any user name and password can be used). Then press OK. A Please Wait window showing an Enumerating Classes message appears. After a few seconds, it is replaced by a Connected To Host window. NOTE: You will see the following error messages if you select the HTTP Transfer protocol. They are harmless and can be ignored. They do not appear when running over RMI.

  • In the left Class Schema pane, folder icons indicate that subclasses exist. To see them, click on the folder icon; e.g. click on the CIM_NextService folder icon. It will now show the CIM_NextServiceAfterMeter subclass.
  • To find a particular class, select Action/Find Class from the top menu. An Input window appears. Enter a class name (e.g. CIM_Process) and press OK.
  • The Connect To Host window is refreshed with CIM_Process highlighted in the left Class Schema pane. Its Properties are shown in the right pane.
  • To see the instances of CIM_Process, select Action/Show Instances from the top menu. A separate Instances for CIM_Process window appears. The left pane shows No instances for CIM_Process because there is no CIM_Process provider. To close this window, select Instance Editor/Exit from the top menu.
  • To see an actual instance, do the following:
      • Select Action/Find Class from the top menu. An Input window appears. Enter WBEMServices_ObjectManagerProviderprotocolAdapter and press OK.
      • The Connect To Host window is refreshed with WBEMServices_ObjectManagerProviderprotocolAdapter highlighted in the left Class Schema pane. Its Properties are shown in the right pane.
      • To see the instances of WBEMServices_ObjectManagerProviderprotocolAdapter, select Action/Show Instances from the top menu. A separate Instances for WBEMServices_ObjectManagerProviderprotocolAdapter window appears. The Instances left pane shows the object path for one WBEMServices_ObjectManagerProviderprotocolAdapter instance.
      • The right pane shows two folder tabs (Properties and Methods). The Properties folder tab is highlighted by default. Here, the properties and values for WBEMServices_ObjectManagerProviderprotocolAdapter are shown. NOTE: Key icons indicate that the property is a Key.
      • To view the methods defined for WBEMServices_ObjectManagerProviderprotocolAdapte, select the Methods folder tab in the right pane. It will now show the StartService and StopService methods.
      • To close the Instances for WBEMServices_ObjectManagerProviderprotocolAdapter window, select Instance Editor/Exit from the top menu.
  • To close the Connect To Host window, select Workshop/Exit from the top menu.

  4. A Browse Documentation by Product page is shown.

How to Run MOF2BEAN on Win32

Last Updated 09/26/03

mof2bean is a command that generates JavaBeans™ that represent the CIM classes defined in a MOF file. mof2bean also needs a Bean configuration file that defines any desired package names, import statements and exceptions that you might want in the generated JavaBean.

WARNING: The MOF file should include any Qualifier definitions that its class definitions require. See Simple.mof

WARNING: The Bean configuration file should not contain any blank lines. See bean.txt

  • Open a command window (DOS window).
  • Go to the directory containing the WBEM Services executables.
    • If you built the WBEM Services executables in c:\files\open, go to directory c:\files\open\dist\wbemservices\examples.
    • If you downloaded or copied the WBEM Services executables into c:\wbemservices, go to directory c:\wbemservices\examples
  • Run mof2bean.bat. You will see the following:

  • The generated .java files will be found in the subdirectory bean; e.g. c:\files\open\dist\wbemservices\bin\bean or c:\wbemservices\bin\bean. Here, the generated files are CIMBeanImpl.java, CIMBean.java, Simple_ClassBeanImpl.java, and Simple_ClassBean.java. CIMBean.java defines the base Java interface while CIMBeanImpl.java represents the base bean that implements this interface. Likewise, Simple_ClassBean.java defines the inteface for the Simple_Class as defined in Simple.mof while Simple_ClassBeanImpl.java represents the bean that implements this interface.

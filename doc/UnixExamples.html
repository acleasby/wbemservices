How to Run Example Programs on Unix

Last Updated 09/26/03

Example programs are provided to show how to develop client applications and providers using the WBEM Services client and provider APIs. Example client programs can be found in the examples/client subdirectory where the WBEM Services executables are located. Example provider programs are found in subdirectory examples/provider. To run some of these example programs to illustrate how the client and provider APIs work, execute the run_examples script.

  • First start the CIMOM by following the directions found here.
  • Go to the directory containing the WBEM Services executables.
    • If you built the WBEM Services executables in /files/open, go to directory /files/open/dist/wbemservices/examples.
    • If you downloaded or copied the WBEM Services executables into /wbemservices, go to directory /wbemservices/examples.
  • Run sh run_examples.sh. You will see the following: 
     wbem% cd /files/open/dist/wbemservices/examples
 wbem% sh run_examples.sh
USAGE: sh run_examples.sh user password [protocol]
   where protocol is rmi (the default) or http

  • run_examples.sh runs example programs to illustrate how some client APIs interact with the CIMOM. It also runs programs to illustrate how client APIs interact with example providers. In these cases, example classes are added to the CIM Repository as needed. These classes are later removed before run_examples.sh terminates. NOTE: The CIMOM will display harmless error messages when running run_examples.sh over HTTP.
